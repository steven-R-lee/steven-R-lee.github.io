Scope in JavaScript

Let's talk scope but first, let's me remind you of what a varaible is in JavaScript parlance, and in computer pragrammming vernacular per se.

A varaible is a container, a general purpose container used in JavaScript. It can hold many items of different forms: numbers, strings, booleans, lists, and more.

what's the scope, then?

The scope indiactes where or where not a particular variable can be seen within a script of JavaScript code. The idea is similar with many other programming languages.

The extent to which a variable is accessible within a script of JavaScript code is determined by the scope of the variable and where that variable is declared. Thus, there are basically two types of scope: Global and Local.

global scope

Global scope is associated with global variables that are declared outside of any function and are therefore accessible from within any function within the script of JavaScript code. These global variables must bare a unique name throughout the entire script of JavaScript code.

local scope

Local scope is associated with local variables that are declared inside a function and are only accessible for use within that function, and nowhere else within the script of JavaScript code. These local variables must bare a unique name only within the function they serve.

one last note

Local variables are generally preferable to global variables as their limited scope prevents possible accidental conflict with other variables.
